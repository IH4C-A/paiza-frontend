# 作業のルール

何か作業をするときはissueを作成してください。
issueを作成する際は、どういう作業を行うのか、かける範囲で説明欄に記入してください。

作業する際のブランチ名は　feature/#issue番号　とかがやりやすいと思います。

# 起動コマンド

npm run dev(今後pnpmを導入する場合、 pnpm run dev になります。)
